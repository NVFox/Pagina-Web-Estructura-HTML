Diagrama de Clases

Muestra la estructura del sistema a través de clases y las relaciones entre estas, teniendo como referencia al paradigma de programación orientada a objetos. Estas clases son mostradas como un grupo de objetos que comparten características y atributos entre sí.

Diagrama de Objetos

Muestra la estructura del sistema a través de instancias concretas de una o varias clases en algún punto de tiempo de existencia del sistema y las relaciones entre dichas instancias.

Diagrama de Paquetes

Muestra la estructura del sistema a través de la representación de los componentes del sistema, por medio de paquetes, que forman parte de la aplicación, y la dependencia entre estos.

Diagrama de Estructura Compuesta

Muestra la estructura del sistema o de partes internas del mismo a través de clasificadores estructurados, que permiten observar clases, paquetes y las relaciones entre todos estos elementos.

Diagrama de Componentes

Muestra la estructura del sistema a través de los componentes que componen el sistema, ya sean de hardware o software, y las relaciones de dependencia entre sí.

Diagrama de Despliegue

Muestra la estructura del sistema a través de la distribución de los componentes del sistema en los nodos físicos (elementos que interactuan con el sistema o forman parte de este) existentes dentro del mismo.

Diagrama de Perfiles

Muestra la estructura del sistema a través de la descripción del metamodelo del sistema. Este metamodelo se refiere a las propiedades del modelo en el que se describe este diagrama.

Diagrama de Casos de Uso

Muestra las funciones que puede hacer un usuario dentro de un sistema, más concretamente, en un caso de uso específico.

Diagrama de Actividad

Muestra el comportamiento y funcionamiento de un determinado proceso u actividad en un sistema de forma secuencial, mostrando flujo de control u objetos.

Diagrama de Máquina de Estados

Muestra el comportamiento de una parte del sistema a traves de transiciones finitas de estados en el sistema.

Diagrama de Interacción

Muestra el comportamiento de un sistema a través de las interacciones entre cada uno de sus elementos.

Diagrama de Secuencia

Muestra el proceso de intercambio de mensajes e interacciones entre objetos del sistema para llevar a cabo una funcionalidad.

Diagrama de Comunicación

Muestra las interacciones entre los objetos o partes (también conocidas como lineas de vida) a través de mensajes intercambiados de forma secuencial.

Diagrama de Tiempos

Muestra los cambios dados en las condiciones que afectan a una línea de vida a largo de un tiempo determinado en la ejecución de ciertos eventos.

Diagrama Global de Interacciones

Muestra el flujo general de control de interacciones y actividades durante la ejecución del sistema. Este flujo puede darnos mayor control y navegabilidad entre los distintos diagramas de Interacción.
